# 한국어 소설 문장 자연스러움 하네스

## 목적

이 문서는 QFUDS SAGA의 한국어 primary 산문이 AI식 번역투·기획서투·비문으로
무너지는 것을 막기 위한 문장 하네스다. 목표는 "AI 탐지 회피"가 아니다.


## 결론: 중간 번역투 금지

SAGA 한국어 산문은 처음부터 한국어 문장으로 설계한다. 외국어식 로그라인, 명사구,
선언문을 만든 뒤 한국어로 옮기는 방식을 기본값으로 두지 않는다.

이유:

- 한국어 소설의 자연스러움은 조사, 어미, 생략, 어순, 호흡에서 나온다.
- 중간 표상을 외국어식 명사구로 두면 한국어 어미와 조사 선택이 뒤늦게 붙어
  번역투가 되기 쉽다.
- 한자어·영어 craft term 압축은 개념 정리에는 도움이 될 수 있지만, 인물의 감각·대사·
  행동을 쓰는 데에는 명사구 과밀과 선언문을 늘릴 수 있다.
- 토큰 효율보다 중요한 것은 target-language prior다. 한국어 산문은 처음부터
  한국어 문장으로 설계한다.

허용되는 예외:

- 용어 후보를 한자어로 짧게 비교할 때.
- 제도명·법률명·분파명처럼 압축 명칭을 고를 때.
- 외국어권 독자용 별도 각색판을 만들 때.

## 한국어 원문 모드

active Korean-primary prose를 쓸 때는 아래 순서를 따른다.

```text
장면 의도 -> 한국어 문장 초안 -> 낭독/호흡 패스 -> 조사/어미 패스
-> 비문 제거 -> 소설성 패스 -> technical fidelity 확인
```

외국어식 로그라인을 먼저 만든 뒤 한국어로 옮기지 않는다. 필요하면 구조 메모만 craft
term으로 남기고, 실제 문장은 처음부터 한국어로 쓴다.

## AI식 한국어 실패 신호

| 실패 신호 | 왜 어색한가 | 고치는 방향 |
| --- | --- | --- |
| `엔진:`, `핵심은 A가 아니라 B`, `사건의 주어는 X` | 기획 회의 문장이 소설/기준문으로 남음 | 선택, 행동, 비용을 직접 쓴다 |
| `그것은 ~의 언어/증명/선언이었다` | 추상명사로 감정을 닫음 | 누가 무엇을 보고 무엇을 했는지 쓴다 |
| `단순히 ~을 넘어` | 발표문·칼럼 리듬 | 비교를 지우고 장면 결과만 남긴다 |
| `~라는 사실을 깨달았다` 반복 | 인물 내면을 요약함 | 깨달음 전후 행동 차이를 보인다 |
| `세계/시대/문명/역사`가 주어 | 사람이 사라짐 | 주어를 인물, 기관, 사물, 양식으로 낮춘다 |
| 한 문장에 추상명사 3개 이상 | 한국어 호흡이 죽고 논문처럼 읽힘 | 명사 하나를 동사나 물건으로 바꾼다 |
| 영어식 `A가 B를 author/produce/encode` 직역 | 한국어 동사 선택이 어색함 | 쓰다, 만들다, 남기다, 정하다, 묶다, 넘기다 등 문맥 동사로 고친다 |
| 같은 어미 `-했다/-였다/-있었다` 연속 | 문장 박자가 평평함 | 짧은 문장, 동작문, 대사, 무주어문을 섞는다 |
| 개념을 작가가 독자에게 해설(`X는 ~이다` 튜토리얼) | 설명용 장면이 되어 몰입이 깨짐 | 인물이 화면·경고문·서식·거래 미리보기를 읽는 장면으로 환산. 정확한 개념은 보존 |
| 유치한 비유(`모두가 들여다보는 장부`, `열쇠가 깎여 나온다`) | 눈높이를 과하게 내려 첫 진입 신뢰가 깨짐 | 한 단계 차갑게(`주소만으로 열쇠의 모양을 좁힐 수 있다`) |
| 작위적 단정문(`기계는 그 거꾸로를 한다`) | 억지 결정문이 번역체처럼 들림 | 세계·인물의 언어로(`그 문장을 폐기한다`). 온보딩 톤 가드는 [010](reader_onboarding_harness.md) |

## 한국어 문장 수술 규칙

### 1. 주어 낮추기

나쁜 방향:

```text
세계는 되돌릴 수 없음이라는 믿음 위에 서 있었다.
```

고치는 방향:

```text
사엘은 복구 불가 판정 도장을 하루에 서른 번 찍었다.
그 도장이 없으면 보험금도, 장례도, 상속도 멈췄다.
```

### 2. 명사구를 동사로 풀기

나쁜 방향:

```text
그 결정은 기준 작성권의 비가역적 이전이었다.
```

고치는 방향:

```text
사엘이 승인하자 다음 양식부터 검증자 서명이 필수 칸으로 들어갔다.
```

### 3. 깨달음을 행동으로 바꾸기

나쁜 방향:

```text
그는 자신이 세계의 반박권을 넘겼다는 사실을 깨달았다.
```

고치는 방향:

```text
사엘은 반려 버튼을 눌렀다. 화면은 접수를 취소하지 않았다.
아래쪽에 그의 전자서명이 남아 있었다.
```

### 4. 어미 단조로움 깨기

한 단락 안에서 `했다/였다/있었다`가 4번 이상 반복되면 낭독 패스를 돌린다.

대체 수단:

- 짧은 동작문.
- 대사 한 줄.
- 무주어 한국어 문장.
- `-자`, `-는데`, `-고도`, `-면` 같은 연결 어미.
- 명사 종결은 아주 드물게만.

### 5. 조사 점검

한국어 문장은 정보 구조가 조사로 갈린다.

| 조사 | 용도 | 점검 |
| --- | --- | --- |
| 은/는 | 대비, 주제 | 너무 많이 쓰면 해설문이 된다 |
| 이/가 | 새 정보, 초점 | 인물 행동을 세울 때 우선 |
| 을/를 | 실제로 영향을 받는 대상 | 추상명사보다 물건/사람에 붙인다 |
| 에/에서 | 장소·상황 | 장면의 바닥을 만든다 |

## 낭독 패스

한국어 primary draft는 최소 한 번 낭독 기준으로 본다.

체크:

- 숨이 모자라면 문장을 자른다.
- 입에 안 붙는 한자어 연쇄는 하나 이상 푼다.
- 조사 없이 붙은 명사열은 끊는다.
- 대사가 실제 사람 입에서 안 나오면 지문으로 빼거나 삭제한다.
- 문단 마지막이 매번 격언처럼 닫히면 마지막 문장을 지운다.

## SAGA 적용 규칙

- QFUDS 개념을 설명하기 전에 인물 손에 든 양식, 도장, 화면, 키, 서명을 보인다.
- `기준 작성권`, `마지막 반박권`, `상실의 저자` 같은 테마어는 design 문서에 남기고
  prose에서는 절차와 선택으로 바꾼다.
- technical term은 숨기지 않는다. 대신 바로 옆에 한국어 손잡이를 준다.
- 판결문, 고지문, 시장 화면, 감사 로그는 건조하게 쓴다. 지문이 대신 장엄해지면
  둘 다 죽는다.

## Prompt Rule

한국어 prose를 요청할 때는 에이전트에 아래를 명시한다.

```text
한국어로 먼저 생각하고 한국어로 쓴다.
외국어식 로그라인을 한국어로 번역하지 않는다.
문장마다 주어를 사람/기관/사물로 낮춘다.
추상 요약문은 장면 행동으로 바꾼다.
낭독했을 때 안 붙는 문장은 실패로 본다.
```
