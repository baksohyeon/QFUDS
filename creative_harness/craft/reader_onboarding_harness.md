# SF 독자 온보딩 하네스

## 목적

독자가 낯선 개념을 모르더라도 현재 장면의 선택, 위험, 감정, 인과를 따라오게 한다.
설명을 금지하지도, 모든 설명에 쉬운 비유를 강제하지도 않는다.

```text
정확성
!= 설명량
!= 독자가 지금 알아야 하는 양
```

작품별 목표 독자와 설명 전략은 work README와 style packet에서 고른다.

## Core Gate

개념마다 한 가지만 먼저 묻는다.

> 독자가 이 개념을 몰라도 지금 누가 무엇을 잃고 어떤 선택을 해야 하는지 이해하는가?

`아니오`라면 아래 전략 중 가장 짧고 자연스러운 것을 고른다. 모든 전략을 한 번에
쓰지 않는다.

| 전략 | 사용 시점 | 위험 |
| --- | --- | --- |
| 직접 서술 | 배경을 짧고 정확하게 압축하는 편이 빠를 때 | 작가 강의로 길어짐 |
| 문제 발생 | 고장, 피해, 악용으로 규칙을 먼저 체감시킬 때 | 원인을 너무 오래 숨김 |
| Incluing | 행동·소품·관용어로 독자가 규칙을 복구할 수 있을 때 | 단서가 부족하면 confusion |
| 인월드 문서 | 로그, 판결문, 경고, 매뉴얼 자체가 세계와 갈등을 드러낼 때 | 위키·설정집처럼 읽힘 |
| 전문가 대화 | 서로 다른 목표를 가진 전문가가 실제로 논쟁할 때 | 독자를 위한 AYKB 대화 |
| 반복 노출 | 처음에는 손잡이만 주고 여러 장면에서 뜻을 쌓을 때 | 초기 오독 가능성 |
| 비유 | 정확한 차이와 한계를 짧게 붙잡게 할 때 | 유치하거나 기술을 왜곡함 |
| 지연 | 지금 몰라도 되는 mystery일 때 | 필요한 정보를 미스터리로 위장함 |

## Information Unit

낯선 개념이 장면에서 처음 중요해질 때 기록한다.

| 항목 | 답 |
| --- | --- |
| Technical / institutional fact | 실제로 맞아야 하는 최소 사실 |
| Needed now | 지금 모르면 어떤 선택이나 인과를 오해하는가 |
| Can wait | 뒤로 미뤄도 되는 사실 |
| Delivery | 위 전략 중 하나 또는 둘 |
| POV reason | 왜 이 인물이 지금 이 사실을 의식하는가 |
| Visible consequence | 행동, 손실, 비용, 권리, 시간, 관계 중 무엇이 달라지는가 |
| Misread risk | 독자가 잘못 추론할 가능성 |

## 실패 신호

- 이미 아는 인물끼리 독자 때문에 규칙을 다시 설명한다.
- 개념을 제거해도 장면의 선택과 결과가 똑같다.
- 용어만 늦게 설명해 놓고 incluing이라고 부른다.
- 독자가 이미 추론한 내용을 narrator가 다시 해설한다.
- 정확한 비유 하나를 넣느라 tone과 인물 voice가 유치해진다.
- 기술어를 전부 숨겨 전문가 독자에게 작동 방식이 거짓으로 보인다.
- 설명은 끝났는데 인물의 선택지나 독자의 예상이 달라지지 않는다.

## Mystery와 Confusion

| 상태 | 독자 반응 | 판정 |
| --- | --- | --- |
| mystery | 무엇인지 궁금하지만 장면의 행동과 위험은 이해한다 | 허용 |
| productive uncertainty | 두 해석이 가능하고 둘 다 다음 장면을 읽게 한다 | 허용 |
| confusion | 누가 무엇을 왜 하는지 복구할 수 없다 | 수정 |
| false clarity | 쉬운 비유는 이해했지만 실제 규칙을 잘못 배웠다 | 수정 |

## First 10% Gate

첫 10%에서는 독자가 아래를 말할 수 있어야 한다.

- 누구의 경험을 따라가는가?
- 지금 무엇이 불안정한가?
- 현실과 다른 최소 한 가지는 무엇인가?
- 당장 몰라도 되는 질문은 무엇인가?
- 지금 알아야 하는 규칙은 무엇인가?

첫 장면이 반드시 액션이나 손실로 시작할 필요는 없다. 하지만 독자가 발을 디딜
장소, 관점, 방향은 있어야 한다.

## 정확성 검증

`plot-critical` 개념은 초고 전에 검증한다. `texture-only` 개념은 기능 placeholder로
쓴 뒤 구조가 고정되면 필요한 질문만 검증한다. 외부 사실을 썼다면 출처 URL,
workflow state, allowed claim, blocked claim을 기록한다.
