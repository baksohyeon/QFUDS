# Plan 20-01: Normalize SAGA shelves and verify harness dry-run

## Objective

Make the fiction writing harness, agentic workflow docs, and QFUDS SAGA internal
folder structure agree with each other.

## Scope

In scope:

- Rename SAGA internal shelves to normalized work-local paths.
- Update active wiki indexes, routing workflow, fiction IP workflow, work README
  template, and operator guide.
- Add empty `30_revisions/` and `40_release/` indexes.
- Add a next Korean-primary prose task gate so agents do not invent the next
  scene without choosing Arc Two planning, first-arc polish, or explicit `025`
  draft creation.
- Run sub-agent checks for harness consistency and dry-run usability.

Out of scope:

- Writing new prose.
- Rewriting existing Korean or English drafts.
- Updating historical `.planning` provenance paths unless they block validation.
- Changing QFUDS research status.

## Acceptance Criteria

- Active docs route work-local harness notes to `00_workroom/`.
- Active docs route canon/reference material to `00_bible/`.
- Active docs route outlines and packets to `10_story_design/`.
- Active docs route prose drafts to `20_drafts/`.
- Active docs route release-facing revision work to `30_revisions/`.
- Active docs route release candidates to `40_release/`.
- Sub-agent dry-run findings are addressed or recorded.
- Validation, workflow guard, preflight, and pre-commit pass before commit.
