# Missing Physics Map

## Status Boundary

This checkpoint documents what is missing before any physical QFUDS branch can
be opened. It is not a roadmap update. Current project status remains governed
by [docs/05_next_steps/000_roadmap.md](../../05_next_steps/000_roadmap.md).

## Current Retained-Branch Classification

Current retained-branch classification:
Phenomenological IV/IDE; Not Yet A Physical Model.

This classification applies to the current retained branch only. It does not
define the entire QFUDS research program.

Physical Level 2B remains blocked.

## Admission Rule

No new physical-QFUDS branch should be opened unless one candidate mechanism
supplies all five items:

```text
X =
Q^nu =
why phase B has w ~= -1 =
delta Q route =
known-model distinction =
```

A partial answer, analogy, fitted timing profile, or renamed coupling function
is not enough.

## Gamma(a) Status

The current Gamma(a) should be treated as a prototype phenomenological transfer
profile, not a derived physical law.

It was useful as a probe because it exposed timing constraints and ruled out
early-time transfer profiles, but it does not by itself explain the physical
origin of phase transfer.

Do not require the current Gamma(a) to be proven.

Instead, future work must decide whether Gamma(a) should be:

- retained only as a phenomenological IV/IDE timing prior;
- replaced by a derived source function `Gamma(a) = f(X)`;
- lifted into a covariant transfer structure `Q^nu`;
- constrained by a perturbation-level `delta Q` prescription;
- or discarded.

## What Would Make Gamma(a) More Rigorous?

The routes below are possible audit routes only. Listing them does not adopt any
route, open Level 2B, or promote retained `Gamma(a)`.

## Source-Scalar Route

Possible form:

```text
Gamma(a) = gamma_0 * X(a) / X_0
```

Required equation, observable, or falsifiable test:

- define `X`, its units, normalization, and physical source;
- show how `X` determines energy transfer instead of merely matching timing;
- predict timing, sign, or amplitude before fitting;
- fail if `X` cannot be measured, computed, or linked to stress-energy.

## Collapse-History Route

Possible form:

```text
Gamma(a) proportional to dF_coll/dln(a)
```

This route is admissible only if `F_coll` is self-consistently computed and
physically connected to energy transfer.

Required equation, observable, or falsifiable test:

- compute collapse history without circularly importing retained `Gamma(a)`;
- derive why collapse changes phase-A and phase-B stress-energy;
- predict how the transfer changes with mass threshold and cosmology;
- fail if the route remains only a timing proxy.

## Black-Hole-Coupled Route

Possible source:

```text
black-hole production or mass growth history
```

Required equation, observable, or falsifiable test:

- define a black-hole source scalar and stress-energy transfer;
- show why the output is smooth vacuum pressure rather than radiation, heat, or
  compact remnants;
- compare directly to black-hole-coupled dark-energy models;
- fail if the route is only an analogy to black-hole information storage.

## Horizon/HDE Route

Possible replacement:

```text
replace Gamma(a) with a horizon-scale energy density relation
```

Required equation, observable, or falsifiable test:

- specify the horizon or IR cutoff relation;
- test whether the result reduces to known HDE or interacting vacuum;
- identify any distinct DM-to-DE phase-transfer prediction;
- fail if the route is only standard HDE with QFUDS labels.

## Covariant Route

Possible replacement:

```text
replace background Gamma(a) with Q^nu
```

Required equation, observable, or falsifiable test:

- derive background and perturbation equations from the same `Q^nu`;
- specify transfer frame, source dependence, and `delta Q`;
- test stability before any observational claim;
- fail if `Q^nu` only restates the background coupling.

## Effective-Action Route

Possible derivation:

```text
derive both phases and their coupling from an action or effective field theory
```

Required equation, observable, or falsifiable test:

- define fields or effective variables for both phases;
- derive stress-energy, equations of state, and coupling terms;
- compute perturbations and stability conditions;
- fail if the action reduces to a known scalar-field, k-essence, unified-fluid,
  or interacting-vacuum model without a new prediction.
