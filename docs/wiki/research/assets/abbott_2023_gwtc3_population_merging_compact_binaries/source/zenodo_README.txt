# GWTC-3 population data release notes

Welcome to the GWTC-3-population data release, associated with the LIGO-Virgo-KAGRA (LVK) paper: 
"The population of merging compact binaries inferred using gravitational waves through GWTC-3" 
[LIGO:P2100239](https://dcc.ligo.org/LIGO-P2100239/public),
[arXiv:2111.03634](https://arxiv.org/abs/2111.03634),
[PRX](https://doi.org/10.1103/PhysRevX.13.011048) 

This is data release v3, [DOI](https://doi.org/10.5281/zenodo.11254021), which fixes some URLs that changed.
No results have changed since v2.  You can always find the latest release at [this DOI](https://doi.org/10.5281/zenodo.5650061).

This is a rather large (6.2 GB) and complex data release. 
If you see any problems, please [alert us](mailto:alan.weinstein@ligo.org) ; thank you!

## Description

- Tables I, II, III, IV, XV, and XVI contain numerical data (the others only contain textual descriptions of the priors used). 
  The files in `table_data.tar.gz` are csv files that can be read in by programs like Excel and Google Sheets, or by Python
  functions like `numpy.genfromtxt` and `pandas.DataFrame`.
  Unpack with `tar xzvf table_data.tar.gz` .

- The Python notebook snapshot `PowerLaw_Peak_DipBreak_Spline_Tutorial.html` is intended to show readers how to load in the data associated with 
  some of the `gwpopulation`-based models used in this paper, and make some plots. 

- The file `paper_figures.tar.gz` contain png or pdf files of all the figures in the paper. 
  Unpack with `tar xzvf paper_figures.tar.gz` .

- The file `GWTC-3-population-data.tar.gz` provides everything you need to reproduce the paper's figures, except for large data files which it will fetch for you.  It contains:
    - `Makefile` for running all of the figure_scripts and auto-fetching large data files.
    - `GWTC-3-population_py_requirements.txt` lists the packages that you will need to install in your python environment (see below).  
        - This includes some LVK-specific software: [`bilby`](https://pypi.org/project/bilby/) and [`gwpopulation`](https://pypi.org/project/gwpopulation/).
    - `tutorials/PowerLaw_Peak_DipBreak_Spline_Tutorial.ipynb` contains the Python notebook mentioned above. 
        - It is designed to be run in the `tutorials/` directory, with a python environment that includes bilby and gwpopulation. 
        - Note that these are example plots, which don't necessarily correspond to the figures in the paper.
    - `figure_scripts/` contains Python scripts for making all the figures in the paper from many different sources of input data.
    - `figures/` is where the figures generated by the Makefile will appear.
    - `utils/` contains some utility Python functions used by multiple figure scripts.
    - `postproc/` contains post-processing files derived from upstream data, needed to make many of the figures in the paper. 
    - `injections/` contains files of signal injections for evaluating search sensitivity.
        - Up to 198 MB of external files may be downloaded here when you run `make`.
    - `o3a_population_data_release/` will contain the data release from the paper: 
      "Population Properties of Compact Objects from the Second LIGO-Virgo Gravitational-Wave Transient Catalog" 
      [arXiv:2010.14533](https://arxiv.org/abs/2010.14533), Published in Astrophys. J. Lett. 913, L7 (2021) 
        - Up to 1.7 GB of external files may be downloaded here when you run `make`.
    - `external_catalog_data/` will contain data releases from other GW catalogs.
        - Up to 89 MB of external files may be downloaded here when you run `make`.
        - Specifically, the external catalogs are:
            - "New binary black hole mergers in the second observing run of Advanced LIGO and Advanced Virgo" [arXiv:1904.07214](https://arxiv.org/abs/1904.07214), Published in Phys. Rev. D 101, 083030 (2020)
            - "3-OGC: Catalog of gravitational waves from compact-binary mergers" [arXiv:2105.09151](https://arxiv.org/abs/2105.09151), Published in Astrophys. J. 922, 76 (2021)
    - `neutron_star_data/` will contain assorted data used in neutron star-containing plots
        - Up to 50 MB of external files may be downloaded here when you run `make`.
        - Specifically, the data come from:
            - "GWTC-1: A Gravitational-Wave Transient Catalog of Compact Binary Mergers Observed by LIGO and Virgo during the First and Second Observing Runs" [arXiv:1811.12907](https://arxiv.org/abs/1811.12907), Published in Phys. Rev. X 9, 031040 (2019)
            - "Impact of the PSR J0740+6620 radius constraint on the properties of high-density matter" [arXiv:2106.05313](https://arxiv.org/abs/2106.05313), Published in Phys. Rev. D 104, 063003 (2021)
            - "A Population-Informed Mass Estimate for Pulsar J0740+6620" [arXiv:2005.00032](https://arxiv.org/abs/2005.00032), RNAAS 4, 65 (2020)

- There are numerous large files with names like analyses_*.tar.gz .
These files will be auto-downloaded if you run the figure-generating Makefile described above,
following thew instructions given below. You will need around 7 GB of free disk space.

## Instructions for generating all the figures from the paper

- Download GWTC-3-population-data.tar.gz into your chosen working directory and un-tar it
  (`tar xzvf GWTC-3-population-data.tar.gz`). 
  This will create the directory (folder) `GWTC-3-population-data`, and all the sub-directories listed above. 
  After that, you can `rm GWTC-3-population-data.tar.gz` if you like.

- All of the following commands should be run inside of the `GWTC-3-population-data` directory.

- First set up your python environment, including several LVK-specific software packages. If you are using `conda`:
```
# prevent conda from picking up elements of your python environment that might interfere
unset PYTHONPATH
export PYTHONNOUSERSITE=True
# Create the conda environment
conda create -n GWTC-3-population
conda activate GWTC-3-population
conda install python==3.9
pip install -r GWTC-3-population_py_requirements.txt
```

- If you close out of your session, in future sessions you just need to run:
```
unset PYTHONPATH
export PYTHONNOUSERSITE=True
conda activate GWTC-3-population
```

- Now you can run the `Makefile` in one of several ways in order to:
    - Generate figures from the paper using the publicly-available data
    - Auto-download and extract the data needed to make those figures

- Generating figures requires numerous data analysis products (data files) that are rather large, 
  totalling 6.2 GB. These have been packed into one tarball per analysis, and will be auto-downloaded
  only as needed:

- The figures will be saved in the `figures/` directory, and they should be _very similar_ to what's in the paper, but not exactly the same.
    - The differences are because many figure scripts choose PE samples randomly, and when run for the paper, they didn't specify a random number seed.

- If things are set up right, `make` shouldn't display any errors, but it *will* display numerous warnings.  They can be ignored.

- If you would like to generate _all_ figures, downloading 6.2 GB of data in the process, just run `make`
    - This will take a long time to run, so if you have a multi-core computer, you can instead run `make -j 4` to use 4 cores (or however many you want).

- If you only want to generate _some_ figures, you can list their names at the end of the make command.  For example:
    - `make FIGURE_1` to generate Figure 1 (will be saved as `figures/O3a_vs_O3b_blob_comparison.pdf`)
    - `make FIGURE_1 FIGURE_2 FIGURE_3` to generate Figures 1, 2, and 3
    - `make FIGURE_13A FIGURE_13B` to generate the two panels of Figure 13, or `make FIGURE_13` as a shorthand


- Note that some figures are generated by the same script, so you may generate one figure and find others were created in the process.

- If you don't want to generate any figures, and just want to download the data for a particular analysis, you can run `make analyses/[NameOfTheAnalysis]`.  For example:
    - `make analyses/PowerLawPeak` to download and extract the data products from the Power Law + Peak model
    - `make analyses/AdaptiveKDE analyses/BinnedGP analyses/Vamana` to download and extract the data products from the Adaptive KDE, Binned GP, and Flexible Mixtures models
