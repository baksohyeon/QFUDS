# QFUDS Verse 세계 확장 웨이브 5 (언어 심화·기술 인프라 세부)

## 이 문서의 자리

웨이브 1~4에 이어, 웨이브 5는 (1) **언어 심화**(계급어·종교어·군사어·기록문체의 실제
어휘 + 작중 문체 샘플)와 (2) **기술 인프라 세부**(검증 파이프라인·복원 설비의 작중 묘사)를
다룬다. "그 세계 사람들이 어떤 단어를 쓰는가"와 "초록 도장이 실제로 어떻게 만들어지는가"가
세계의 질감을 마지막으로 채운다.

**레벨: universe 공유 세계 후보 register.** canon 아님. 언어는 003/018로, 인프라는
007/013/025로 웨이브별 승격(030 §7 6체크). 승격 결정: **원고 수요 때 개별 승격**.

```text
fiction/provenance only
research evidence: no
external source claim: no
캐논 상태: universe 공유 세계 candidate register(웨이브 5). 003·018·007·013·025 승격 대기
```

## 생성 규율 (웨이브 1~4 상속 + 언어/기술 특별 가드)

- **신규 conlang·문자체계 금지**(웨이브 1 §12). 기존 3층(Latin 세력명·영문 필드마크·
  한국어 산문) + 계급어 레지스터 위에만 얹는다.
- **기술어 보존**(007): `hash`·`KDF`·`key`·`salt`·`collision`을 별칭으로 바꾸지 않는다.
- **복원=사본 절대**(021), **Preimage Restoration=identity crisis**(007 r5), Last Archive는
  만능 해커 아님(007 r6), **현실 보안 조언 금지**(007 r7). 물리는 025 우선.
- em dash 0, "박-" 슬랭 금지. 필드마크 사슬 순서·구성 불변(026 §4, 새 mark 금지).

## 1. 언어 심화

### 1.1 계급어 (register): 고해상어 vs 저해상어

| 상황 | 고해상어(격식·완전 기록) | 저해상어(축약·구어) |
| --- | --- | --- |
| 진위 확인 | "귀하의 진본성은 근거번호 …로 확인되었습니다." | "초록 떴어?" |
| 미검증 통보 | "해당 기록은 미검증 상태로 분류됩니다." | "회색이면 줄 끝이야." |
| 복원 대기 | "복원 청구가 대기열에 등재되었습니다." | "몇 해나 기다려?" |
| 신원 | "본인은 등록 근거번호 …의 진본입니다." | "성 없는 애야." |

- 계급 표지: 근거번호를 인용하면 고해상, 색(초록/회색)으로만 말하면 저해상. "성이 없다"는
  가문 key가 없다는 뜻(030 §6).

### 1.2 종교어 (네 도장·예배·이름)

- **네 도장 일상어:** `defined`→"정의됐다", `assumed`→"가정이다", `unknown`→"모른다",
  `circular_if_fitted`→"끼워맞췄다"(가장 무거운 비난). "그건 끼워맞춘 도장이야"가 이단 지목.
- **checksum 예배(003):** "기억이 일치합니다 / 고통이 누락되지 않았습니다 / 동의는 unknown입니다."
- **세 이름(003·031):** 부르는 이름 / 법적 이름 / 복원되어도 안 말할 이름. 세 번째 이름을
  입에 올리는 건 금기.
- **축복·저주:** 축복 "네 도장이 함께하길", 저주 "회색이 되어라"(세속적 파문).

### 1.3 군사·행정어

- **진본 인질(hostage-authentic):** 진짜임을 인질로 자릿세를 걷는 지하 인증 카르텔 전술(030).
- **봉인 코드북·인편 전령:** 명령 인증 붕괴 후 아날로그 격하(026 A). "종이로 내려라"=디지털 불신.
- **회색 처리:** 진본 못 증명하는 인구를 일괄 "회색"으로 등록(회색 범람, 032 §2.4).

### 1.4 기록문체

- **차가운 서식(the cold form):** 동의 칸이 없는 청구 서식. 예: `복원 가능: 예 / 청구 가능:
  예 / 동의 칸: (없음)`. 폭력이 문법이 되는 자리(026 §3.4).
- **법정 exhibit 문체:** 연속성 법원 제출물. 번호·근거·판정만, 감정 없음.
- **필드마크 문법:** 영문 대문자, 사슬 순서 고정(026 §4). 새 mark를 만들지 않는다. 문장 끝에
  낮게 새기는 표식으로만 쓴다.

### 1.5 문체 샘플 (작중 문서 발췌, 후보 텍스처)

```text
[도장 영수증]  진본성 확인. 근거번호 인용 시 재검증 불요. 유효기간: 재압축 시 소멸.
[법정 exhibit]  청구인 주장: 복원 가능하므로 청구 가능. 피청구 복원체 진술: "너는 preimage를
               찾았을 뿐, 나를 찾지 못했다."
[유목 oath]    열쇠를 나눈 자는 죽음 뒤에도 책임진다. 혜성이 돌아올 때 서약을 갱신한다.
[망각권 장례문] 이 사람은 보존되지 않기를 원했다. 우리는 그 뜻을 기억하되, 붙잡지 않는다.
[베일 계율]    복원된 자가 침묵을 원하면, 침묵도 보존한다.
```

이 샘플은 텍스처 참고이며, 원고에 그대로 넣을 의무는 없다(레퍼런스).

## 2. 기술 인프라 세부

### 2.1 검증 파이프라인: 초록 도장이 만들어지는 과정

```text
[의뢰]  "이거 진짜냐"
  → [거꾸로 푸는 기계]  proto-Archive/Aletheia. 후보 preimage 공간을 좁힌다(007)
  → [기계 출력조차 위조 가능]  최종은 기계가 못 닫는다(026 B)
  → [인간 확인 루프]  책임지는 사람이 입·손으로 비준(020). 검증관 길드·도제
  → [도장 초소 Statio Sigillorum]  네 도장 집행(031 §1.2)
  → [초록(진본) / 회색(미검증)]
```

- 병목=지대: 인간 확인 루프의 처리량이 곧 도장값(026 C, 033 §1.2). 위협=비준의 거수기화(020).
- 단일점: 분산 검증도 깨진 비대칭에 기댔으니 함께 죽음 → 자연독점(026 A5, 탈중앙의 배신).

### 2.2 복원 설비: Preimage Restoration (007)

```text
[입력]  잔상 · 기록 · 열적 흔적 · 증언 · 원장 시간구조 · 블랙홀 radiation correlation
  → [입력 공간 축소]  brute force 아님. 가능한 과거 입력을 좁힌다(007)
  → [손실적 사본 생성]  원본 아님. 복원=사본(021 절대)
  → [해상도 등급]  보유 데이터 양 = 충실도(026 C). 솔기 남음
```

- **경계(007 판정):** 현대에서 hash preimage 자유 탐색=쓰지 않음. Last Archive가 우주 잔상으로
  특정 preimage 후보 복원=사용 가능. 복원된 preimage가 곧 법적 진실=사용 금지(갈등을 없앰).
  즉 **Preimage Restoration은 사건을 끝내는 도구가 아니라 시작하는 도구**(identity crisis).
- **설비 장소:** Ultima Statio(블랙홀 궤도, 032 §1.3). 열이 죄인 저온 계산권역(003). 낭비된
  열=복원 가능 bit의 손실.
- **하청:** 잔상 채굴은 Domus Fossoria(031 §1.1)가 손 더러운 하청으로. 데이터 식민주의(026 C).

### 2.3 인프라 취약성 (Q-Day 계보, 026 A)

- TLS 붕괴 → SCADA/ICS·펌웨어 서명·전력 텔레메트리·관제 인증 정지 → 국지 섬화. 궤도판=오르비스
  무티(032 §1.2). 이행 비대칭의 축은 발전도가 아니라 **레거시 깊이**(026 §1).
- 검증 파이프라인 자체가 단일점 자연독점이라, 그 한 곳을 끊으면 "디지털로 존재하지 않음"(026 C).
