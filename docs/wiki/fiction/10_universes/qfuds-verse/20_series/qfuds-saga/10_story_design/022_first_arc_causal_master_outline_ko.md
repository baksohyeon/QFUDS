# QFUDS SAGA 1부 통합 causal master outline

## 1. 문서 역할과 boundary

016(outline)·017(scene cards)·020(Q-Day bridge)·021(reveal ladder)에 흩어진 결정을
**한 줄의 사건 인과**로 합친다. 각 비트가 카드가 아니라 *원인 → 결과*로 이어지는지
잠그고, B1 압축·B2 산문·Aletheia bible 승격의 선후를 결정하는 기준 문서다.

boundary: fiction/provenance only, QFUDS 연구 증거 아님. 산문 아님. **새 세계관 설정을
만들지 않는다**(기존 결정을 실행 가능한 아웃라인으로 통합만). 모순은 "패치 필요"로
표시만 한다. cascade·030·029 수정은 범위 밖.

## 2. 1부 전체 한 줄 인과

```text
Q-Day가 소유 증명(유효한 서명 = 진짜 주인)을 붕괴시킨다.
→ 기존 기관은 온체인/오프체인 증거가 파편화돼 단독 판정하지 못한다.
→ Aletheia가 분쟁 증거를 통합하는 임시 검증 API(Provisional Ownership Seal)로 호출된다.
→ 봉쇄로 상급자가 없는 밤, 최말단 검증관 사엘이 한 지갑 사건에 provisional seal을 찍어
   한 사람을 구하고 최초 선례를 만든다.
→ 그 선례가 큐로 폭증하고, Aletheia seal 없이는 동결·환급·보험·소유 이전을 처리할 수
   없게 되어, 돈이 아니라 분쟁·증거·판정권이 Aletheia로 흡수된다.
→ "복원 가능한 것은 청구 가능하고, 무엇이 진짜인지는 검증자가 정한다"는 문장이
   지갑 → 기록 → 사람으로 올라가, 첫 사람-청구 대상 Mara로 2부에 넘어간다.
```

## 4. Q-Day → Aletheia 호출 인과

1. **표면 사건:** 피해 캡처 → 오인(사기·피싱·라자루스·러시아·내부자·정부·앱버그) →
   거래소 출금 정지 → 장부·거래·서명 정상 → 피해자 "나는 보낸 적 없다".
2. **기존 기관 한계(증거 파편화):** 거래소=플랫폼 로그만 / 체인분석사=온체인 이동만 /
   정부·법원=느리고 책임 큼 / 보험사·수탁기관=보상 기준 필요. 각자 *부분 증거*만 쥠.
3. **단독 판정 불가:** 아무도 "유효한 서명 = 진짜 의사인가"를 혼자 판정하지 못한다.
4. **evidence aggregation 수요 발생** → 흩어진 증거를 합칠 통합 검증자가 필요.
5. **Aletheia = 거래소가 아니라 B2B 진본성 보안/포렌식/evidence aggregation SaaS**
   (Shield/Trace/Seal/Grid). 원래 위조 탐지·전자서명 검증·체인 포렌식 회사.
6. **Disputed Transfer Triage / Provisional Ownership Seal이 임시 표준**이 됨.
7. **신뢰가 아니라 의존으로 판정권 획득**(책임 회피 + 처리 편의 → 임시 표준 → 의존 →
   사실상 판정권). 돈이 아니라 분쟁·증거·판정권이 Aletheia로 흡수된다.

## 5. B2 파일 발생 인과

- **왜 생기나:** 온체인은 유효한데 KYC-linked 이전 보유자가 부인하는 충돌 케이스.
- **왜 거래소/법원/정부가 직접 못 하나:** 각자 부분 증거뿐(§4), 책임이 크고 느림.
- **왜 Aletheia로 오나:** 증거를 통합해 임시 분류(seal)할 수 있는 유일한 지점.
- **왜 사엘에게 내려오나:** 봉쇄로 상급자 부재 + First Pass Intake Clerk 직무에 내장된
  긴급 비준 권한 + 비준 기한(§6).
- **파일 최소 구조:**

```text
ON-CHAIN: valid transaction, valid signature, transfer settled
OFF-CHAIN: KYC-linked prior holder, not authorized claim, session/device mismatch,
           no matching 2FA, anomalous recipient cluster, Q-Day exposure pattern match
REQUEST : provisional ownership seal
QUESTION: Can this valid signature be treated as valid intent?
```

## 6. 사엘 권한과 원죄 인과

- **왜 최말단이 권한을 쥐나:** First Pass Intake Clerk는 1차 접수·분류 자리라, SLA/
  throughput 설계상 잠정(provisional) seal 분류 권한이 직무에 내장돼 있다. 정식 판결이
  아니라 *임시 분류*라서 아랫줄에 열려 있다(이게 비극의 씨앗).
- **왜 사엘이 그 자리에 서나:** 전날 복원권 시위로 세 부서 동시 봉쇄 → 상급자 부재 →
  비준 기한은 흐름 → 위로 올리면 사건이 멈추고 그 사람이 지워짐.
- **원죄 형태(B→A→C):** 막으려다 실패(B) → 한 사람을 구하려고 authentic으로 비준(A) →
  직후 보고/상신 버튼을 닫는 침묵(C).
- **왜 실수가 아니라 선례인가:** 잠정 비준도 seal protocol상 audit record로 영구
  기록되어 후속 사건에서 precedent로 자동 인용된다. "그냥 실수"가 아니라 제도 안에서
  유효한 최초 선례 문장이 된다.

## 7. 선례 확산 인과

- **왜 폭증하나:** 모두가 B4의 도장을 "선례"로 인용해 같은 복원·소유 판정을 요청.
- **왜 seal 없이 처리 불가인가:** 거래소 동결·보험 환급·법원 소유 이전이 전부 "진짜
  주인" 판정을 전제하는데, 그 판정의 유일한 임시 기준이 Aletheia seal이 됨.
- **흡수의 의미:** 돈이 아니라 분쟁·증거·판정권이 Aletheia로 빨려든다(권력의 축적).
- **연출 규칙(B5/B6):** 설명이 아니라 사엘 책상의 화면·큐 숫자·꺼지지 않는 경보·위임
  로그·"provisional seal upheld" 자동 승인으로만 보여준다(021 §8·§9 과밀 금지).

## 8. Mara handoff 인과

- **사다리:** 지갑(소유) → 지워진 기록(데이터) → 마지막 순간 → 사람. 같은 도장, 더 큰
  대상(017 B7, 016 §9).
- **논리 확장:** "유효한 서명 ≠ 진짜 의사"(1부)가 "정밀한 복원 ≠ 원본의 동의·동일성"
  (2부)으로 올라간다. 검증자의 판정권이 돈에서 사람으로 적용 범위를 넓힌다.
- **1부 마지막에 독자가 들고 갈 질문:** 누가 진짜를 판정하는가 / **누가 상실을 쓸
  권리가 있는가**(`who may author loss`, 015 Q5).
