# QFUDS SAGA Mara Veyr 프롤로그 연구 하네스

## 목적

이 문서는 option C, 즉 **Mara Veyr 사건을 프롤로그로 쓰기 위한 작가실
하네스**다.

바로 소설 본문을 쓰지 않는다. 먼저 작가처럼 준비한다.

1. 소설 장면이 굴러가는 구조를 정한다.
2. COVID 이후 디지털 애도, 망각권, AI-generated reality, cyber memorial
   ritual을 사회 배경으로 변환한다.
3. Bitcoin/Genesis Chain, `it from bit`, black-hole/white-hole restoration을
   과학적으로 과장하지 않는 fiction premise로 분리한다.
4. Mara Veyr 사건을 장면 목표, 충돌, 파국으로 압축한다.
5. 이후 실제 프롤로그 초안을 쓰기 위한 agent harness와 승인 gate를 고정한다.

이 문서는 fiction/provenance only다. QFUDS 연구 결과, 물리 증거, support,
validation, Level 2B admission이 아니다.

## Workflow Boundary

이 문서는 외부 글쓰기/법/디지털 애도/AI 영상 참고자료를 craft/background
source로만 사용한다. 새 연구 asset, numerical product, PDF digitization,
QFUDS evidence claim을 만들지 않는다.

외부 자료 handling은
[Research Asset and Product Workflow](../../../../../../../../.agent/workflows/research-asset-product-workflow.md)
가 계속 지배한다.

현재 workflow state:

```text
hit_not_cached
asset_available_not_downloaded
```

아래 source들은 SAGA craft/background 참고다. repo research cache로 다운로드하거나
수치화하지 않았다. fiction 설정을 현실 QFUDS source claim으로 되돌려 쓰면 안
된다.

## Novelist Rule

설정은 장면 뒤에 온다.

프롤로그의 중심은 다음 명제가 아니다.

```text
완전 복원은 인격 동일성과 동의 문제를 만든다.
```

중심은 이 장면이다.

```text
한 남자가 죽은 아내를 되살렸다.
법원은 그것이 사랑인지 범죄인지 판정해야 한다.
복원된 아내는 자신이 아내가 아니라고 말한다.
```

모든 개념은 인물의 행동으로 바꾼다.

| 개념 | 설명으로 쓰면 약함 | 장면으로 바꾸면 강함 |
| --- | --- | --- |
| right to be forgotten | "망각권이 중요하다" | Mara가 자기 이름이 새겨진 복원 명령서에 침을 뱉지 않고 조용히 뒤집어 놓음 |
| cyber memorial ritual | "디지털 장례가 확산됐다" | 법정 방청석에 가족들이 holographic memorial veil을 쓰고 있음 |
| AI reality editing | "AI가 현실을 편집했다" | 법정 exhibit이 고대 밈 자체가 아니라 "안심할 만큼 가짜였던 실패"가 사라진 순간을 증거화함 |
| Bitcoin artifact | "Bitcoin은 고대 artifact다" | 법정 서기가 Genesis Chain block hash를 성물처럼 봉인함 |
| black-hole restoration | "호킹복사 역연산이 가능하다" | Elias의 손목에 event-horizon exposure burn이 남아 있음 |
| white-hole return | "화이트홀은 return channel이다" | 금지 protocol 이름이 `Protocolum Regressus`로 봉인되어 있음 |

## Social Background

Mara Veyr 사건은 먼 미래 재판이지만, 독자가 알아볼 수 있는 현대적 불안을
뿌리로 가져야 한다.

### 0. Before The Pandemic

죽음은 이미 online에 흔적을 남기고 있었다.

고인의 계정, 사진, 메시지, 검색 기록, 위치 기록, 구독 내역은 가족과 플랫폼,
국가와 기업 사이에 남았다. 사람들은 아직 그것을 "afterlife"라고 부르기보다
"계정 정리"라고 불렀다.

### 1. COVID Grief Layer

COVID 이후 장례와 애도는 물리적 공간에서 완전히 분리되지 못했다.

사람들은 화면으로 장례에 참석했다. 격리된 병실에서 마지막 목소리를 들었다.
공중보건을 위해 이동, 접촉, 증상, 위험이 데이터가 되었다. 이 시대는 SAGA에서
중요하다. 복원 문명은 어느 날 갑자기 나타난 것이 아니라, 이미 슬픔과 몸이
데이터화되는 세계에서 자랐다.

이 시대의 후대 명칭:

```text
The Remote Mourning
```

### 2. The Soft Editing

2023-2026 전후 인간은 생성 이미지와 영상을 장난처럼 소비했다.

초기 영상은 이상했다. 손은 무너졌고, 얼굴은 녹았고, 음식은 물리법칙을
배신했다. 사람들은 웃었다. 안심할 수 있었기 때문이다.

하지만 모델은 조용히 나아졌다. 조명, 목소리, 얼굴, camera movement, scene
continuity가 개선되면서 웃음의 종류가 바뀌었다.

```text
예전의 웃음:
기계는 아직 가짜다.

나중의 웃음:
왜 이것이 진짜처럼 보이는가?
```

이 시대의 후대 명칭:

```text
The Soft Editing
```

### 3. Memorial Capitalism

디지털 장례와 memorial page는 산업이 되었다.

처음에는 사진과 댓글이었다. 다음에는 voice clone, griefbot, anniversary
message, 자동 추모 영상이었다. 나중에는 사후 reputation management와 memory
curation 보험이 붙었다.

복원 문명은 여기서 태어난다.

```text
죽은 이를 기억하는 서비스
-> 죽은 이를 대화하게 하는 서비스
-> 죽은 이를 법적으로 대리하는 서비스
-> 죽은 이를 복원하는 서비스
```

### 4. Genesis Chain Archaeology

Bitcoin은 이 사회에서 다시 발견된다.

돈이라서가 아니다. 최초로 세계 규모의 trust를 bit와 합의에 맡겼던 고대
artifact였기 때문이다.

후대 학자들은 Bitcoin을 이렇게 읽는다.

```text
The first people who feared institutions
built an institution out of numbers.
```

### 5. Cryptographic Death

역연산 기술은 기존 암호학의 신화를 무너뜨린다.

모든 암호가 단순히 "한 번에 다 뚫린다"는 식으로 쓰면 유치하다. 더 강한
방식은 사회적 충격으로 쓰는 것이다.

```text
어제까지 private key였던 것이
오늘부터 고고학적 유물이 되었다.
```

이 순간 Bitcoin은 무용해지지 않는다. 오히려 성유물이 된다. 지킬 수 없게 된
왕관은 왕관이 아니라 유골함이 된다.

### 6. Restoration Courts

마지막 단계는 법이다.

완전 복원은 과학 문제가 아니라 재판 문제가 된다.

```text
기억이 같으면 같은 사람인가?
동의서도 복원 가능한가?
죽은 사람이 복원 후에 과거의 거부를 철회할 수 있는가?
복원된 사람이 "나는 원본이 아니다"라고 말하면 누가 이기는가?
```

Mara Veyr 사건은 이 질문들이 처음으로 한 법정에 모인 날이다.

## Agent Harness

이 fiction workflow는 research workflow와 분리한다. 단, 외부 자료를 쓰면
research asset workflow marker를 남긴다.

### Roles

| Agent | Job | Output |
| --- | --- | --- |
| `showrunner` | SAGA 주제와 장기 떡밥 유지 | season spine, canon decisions |
| `craft_editor` | 장면 목표/충돌/파국, show/tell, pacing 점검 | beat sheet critique |
| `social_historian` | COVID 이후 digital mourning, erasure, AI reality-editing 배경 관리 | timeline/context notes |
| `science_auditor` | Bitcoin/암호학/블랙홀/화이트홀/QFUDS 과장 방지 | allowed/forbidden claim table |
| `continuity_auditor` | 003-007 문서 간 명칭/기관/사건 일관성 점검 | canon consistency report |
| `scene_drafter` | 실제 프롤로그 장면 작성 | prose draft |
| `style_editor` | 문장 리듬, 감각 detail, 대사, 과잉설명 제거 | revised prose |

### Workflow

```text
1. Research intake
   -> craft/social/science background only

2. Premise audit
   -> what is fiction, what is real-world background, what is forbidden claim

3. Beat sheet
   -> scene goal, conflict, disaster, final image

4. User gate
   -> user approves POV, tone, names, and first-scene direction

5. Scene draft
   -> 3rd-limited prose with document interlude allowed

6. Red-team pass
   -> remove lecture mode, remove QFUDS evidence leakage, sharpen character desire

7. Revision
   -> prose rhythm, image system, dialogue, ending hook

8. Commit
   -> narrow staged docs, validation, commit
```

### MCP / Tool Candidates

| Tool class | Use | Boundary |
| --- | --- | --- |
| Canon/RAG MCP | 인물, 기관, 사건, 명칭 일관성 검사 | fiction canon only |
| Web/search | craft/background reference scout | source state required |
| PageIndex/MarkItDown | 논문/PDF가 실제로 필요할 때만 | not needed for this draft unless user asks |

## Prologue POV Decision

추천 POV는 **3인칭 제한 시점, Continuity Court auditor**다.

Mara POV로 바로 시작하면 감정은 강하지만 미스터리가 줄어든다. Elias POV로
시작하면 사랑의 집착이 강하지만 독자가 그의 범죄를 너무 빨리 이해하게 된다.

법정 감사관 POV는 세 가지를 동시에 준다.

1. 독자가 이 세계의 법과 의례를 배운다.
2. Elias와 Mara를 모두 의심할 수 있다.
3. 마지막에 Mara의 한 문장이 법정과 독자를 동시에 깨뜨린다.

1인칭은 interlude로만 쓴다.

예:

```text
Exhibit M-17:
Posthumous refusal of restoration, signed by Mara Veyr.
```

## Scene Beat Sheet

### Scene Goal

Elias Veyr는 Mara의 복원이 원본의 귀환임을 법적으로 인정받으려 한다.

### Opposition

Continuity Court는 복원 동의, 원본성, archive exception, public-interest
restoration의 충돌을 심리한다. Keyless Orders는 법정 밖에서 복원 반대 시위를
한다. Ledger Houses는 Genesis Chain artifact와 restoration protocol의 소유권을
방어한다.

### Hidden Opposition

Mara 자신이 가장 큰 opposition이다.

그녀는 자신이 Elias의 아내라는 증거를 모두 통과한다. 그리고 그 증거를
통과한 뒤 자신이 그의 아내가 아니라고 말한다.

### Disaster

법정은 identity test를 통과한 복원체가 self-denial을 할 경우의 판례를 갖고
있지 않다.

더 나쁜 문제:

Mara의 self-denial이 복원 오류가 아니라, 원본 Mara의 사후동의서와 복원된
Mara의 현재 발화가 동시에 참이라는 첫 사례일 수 있다.

## Tone Target

톤은 논문 설명이 아니라 차가운 법정 SF다.

```text
courtroom ritual
+ black-box science horror
+ grieving spouse tragedy
+ archive mystery
```

피해야 할 톤:

- 인터넷 밈 에세이로 길게 설명하기.
- "AI가 발전해서 위험하다" 설교하기.
- Elias를 단순 악당으로 만들기.
- Mara를 단순 피해자로만 만들기.
- QFUDS를 맞는 이론처럼 말하기.

살려야 할 톤:

- Elias가 사랑 때문에 선을 넘었다는 사실.
- Mara가 그 사랑을 이해하지만 용서하지 않는다는 사실.
- 법정이 인간보다 차갑지만, 인간보다 정직할 수도 있다는 느낌.
- Bitcoin/AI/black hole restoration이 모두 "기억을 지우지 못하는 문명"으로
  모인다는 구조.
