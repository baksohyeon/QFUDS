# Wiki Fiction

This folder stores fiction companion material derived from QFUDS provenance,
research process, and public-story records.

It is a separate creative track. It does not change research evidence or roadmap
status.

   - studio/IP management rules for universes, continuity, works, bibles,
   story design, drafts, releases, and archives.

   - how to use GSD planning as execution management without replacing the

## Boundary

Fiction may preserve motivation, emotional continuity, and explanatory imagery.
It must not promote a speculative image into a physical source claim.

Active fiction-IP system notes belong under
[Fiction IP Management System](00_studio/001_fiction_ip_management_system_ko.md).
The current work list and migration gate live in
[Fiction Catalog](01_catalog/README.md). Current active fiction work belongs
under [QFUDS SAGA](qfuds-saga/README.md). The current universe/IP scaffold is
[QFUDS Verse](10_universes/qfuds-verse/README.md).
Superseded or prototype fiction belongs under [Fiction Archive](archive/README.md).

Research-source handling remains governed by
[Research Asset and Product Workflow](../../../.agent/workflows/research-asset-product-workflow.md)
whenever fiction-system work touches an external paper, web reference, PDF,
MCP, code repository, or source/product availability claim.
