# Fiction Templates

These templates support the
[Fiction IP Management Workflow](../../workflows/fiction-ip-management-workflow.md).

Use them for new fiction universes and works before writing prose.

| Template | Use when |
| --- | --- |
| [universe_readme_template.md](universe_readme_template.md) | Creating a new universe/IP under `fiction/worlds/`. |
| [work_readme_template.md](work_readme_template.md) | Creating a new series, novel, short, anthology, webtoon-like run, or elseworld work under `fiction/projects/<work-id>/`. |
| [work_bible_template.md](work_bible_template.md) | Creating a work-local "Work Bible" section in a work's `README.md`, or a universe's `series-bible/` when the bible is shared across works. |
| [character_sheet_template.md](character_sheet_template.md) | Profiling a character that carries a scene or arc (companion to the bible Cast table). |
| [continuity_audit_template.md](continuity_audit_template.md) | Cross-checking a draft against the work bible before revision/release. |
| [session_brief_template.md](session_brief_template.md) | Starting a writing/revision session with user approval gates and boundaries. |
| [gsd_phase_brief_template.md](gsd_phase_brief_template.md) | Planning a bounded GSD phase for fiction/IP migration, scaffolding, writing, or verification work. |
| [fiction_production_board_template.md](fiction_production_board_template.md) | Tracking chapter/release-tier state across drafting, workshop, revision, verification, and release. |
| [chapter_intent_card_template.md](chapter_intent_card_template.md) | Defining unit type, POV, speculative constraint, state change, reveal budget, and story promises before a major unit. |
| [review_wave_protocol_template.md](review_wave_protocol_template.md) | Running foundation scan, high-severity fix, re-scan, continuity fix, voice polish, and release gate in order. |
| [chronicler_pass_template.md](chronicler_pass_template.md) | Recovering canon deltas, character-state changes, hooks, technical terms, and continuity risks after a draft or revision. |
| [style_packet_template.md](style_packet_template.md) | Creating a compact work-local prose style packet for active writing and final polish. |
| [truth_state_ledger_template.md](truth_state_ledger_template.md) | Tracking what characters know, what readers know, what the world confirms, and what contradictions remain open. |
| [zettel_template.md](zettel_template.md) | Distilling a reusable cross-project thought from a brain dump or source. |
| [source_record_template.md](source_record_template.md) | Recording a fiction-only external source, its verified anchor, blocked claim, and extrapolation. |
| [story_seed_template.md](story_seed_template.md) | Combining Zettels into a candidate premise without promoting it to a project or canon. |
| [revision_plan_template.md](revision_plan_template.md) | Turning workshop evidence into a scoped macro-before-polish change plan. |
| [revision_memo_template.md](revision_memo_template.md) | Recording applied, rejected, deferred, and newly created revision issues. |
| [workshop_response_template.md](workshop_response_template.md) | Giving evidence-based, author-centered feedback before a revision plan. |
| [release_checklist_template.md](release_checklist_template.md) | Binding a release decision to one committed candidate and its review evidence. |
